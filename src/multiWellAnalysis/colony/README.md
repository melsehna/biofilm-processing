# Colony-level biofilm segmentation and feature extraction

This module contains the colony-level biofilm analysis pipeline used to
segment discrete biofilm colonies from processed brightfield timelapse
images and extract per-colony and per-well features.

It is designed for **high-throughput phenotyping** of bacterial mutants
imaged in multi-well plates (e.g. Cytation / BioSpa workflows).

---

## Overview

Pipeline stages:

1. Load preprocessed, registered timelapse images  
2. Segment biofilm colonies per frame  
3. Extract per-colony geometric, spatial, and texture features  
4. Aggregate features at the well level  
5. Track biomass and colony statistics over time  

The pipeline emphasizes **robustness and scalability** over pixel-perfect
segmentation, and is intended for comparative phenotype analysis rather
than single-colony reconstruction.

---

## Directory structure

- `segmentation.py`  
  Core colony segmentation logic (thresholding, enhancement, masking)

- `colonyFeats.py`  
  Per-colony feature extraction:
  - geometry (area, eccentricity, circularity)
  - spatial features
  - neighbor / graph-based features
  - texture features (Haralick, etc.)

- `wellAggregation.py`  
  Aggregation of colony-level features into per-well summaries

- `io_utils.py`  
  Shared I/O helpers, logging utilities, and checkpoint handling

- `runSegmentation.py`  
  Single-plate or single-well segmentation runner

- `runBatch.py`  
  Batch runner for processing entire plates or experiments in parallel

---

## Inputs

- Preprocessed timelapse TIFFs  
  Expected shape: `(T, H, W)`  
  Generated by the upstream preprocessing / registration pipeline

- Plate index CSV  
  Must include:
  - `plateID`
  - `wellID`
  - `processed_path`
  - `nFrames`

---

## Outputs

- Per-well colony feature CSVs  

- <wellID>_colony.csv

- Biomass and colony summary time series

- Per-well logs and checkpoint files to support restartable execution

---

## Key design choices

- **Frame-wise segmentation**  
Colonies are segmented independently per frame rather than tracked
explicitly across time.

- **Feature-first philosophy**  
The goal is phenotype discrimination and clustering, not perfect
colony boundaries.

- **Checkpointed execution**  
Wells are skipped automatically if outputs already exist.

- **Parallel-safe**  
Designed for multi-core and cluster execution.

---

## Usage

Typical execution via module invocation:

```bash
python -m multiWellAnalysis.colony.runBatch \
--index plate_index.csv \
--outdir output_dir \
--workers 24
```

For debugging or development, individual wells or plates can be run using
runSegmentation.py.

---

## Notes

- Segmentation parameters are tuned for brightfield biofilm imaging
and may require adjustment for other modalities.

- Early frames may be excluded to avoid seeding and focus artifacts.

- This module is intended to be used before whole-image phenotype extraction and downstream embedding (UMAP, clustering).


## Status 

Actively used and maintained. The API is stable but optimized for internal use within the multiWellAnalysis pipeline rather than as a standalone package.
